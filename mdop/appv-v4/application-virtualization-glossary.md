# Glossaire des applications virtuelles


<table>
<colgroup>
<col width="50%" />
<col width="50%" />
</colgroup>
<thead>
<tr class="header">
<th align="left">Terme</th>
<th align="left">Définition</th>
</tr>
</thead>
<tbody>
<tr class="odd">
<td align="left"><p>mise à niveau active</p></td>
<td align="left"><p>Une mise à niveau permettant d’ajouter une nouvelle version d’une application à un serveur d’administration d’applications ou à un serveur de diffusion en continu sans affecter les utilisateurs exécutant actuellement l’application.</p></td>
</tr>
<tr class="even">
<td align="left"><p>Racine de l’application</p></td>
<td align="left"><p>Référence à une clé de Registre qui permet une substitution du code base OSD pour l’élément HREF (par exemple, l’emplacement source). Cette valeur de registre permet à un administrateur ou système ESD de garantir le chargement d’une application en fonction d’un schéma de gestion de topologie planifié.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Lecteur de virtualisation des applications</p></td>
<td align="left"><p>Le lecteur du client d’application virtuel par défaut (Q:) à partir desquels les applications séquencées sont exécutées.</p></td>
</tr>
<tr class="even">
<td align="left"><p>Rapports sur la virtualisation des applications</p></td>
<td align="left"><p>Informations d’application virtuelles collectées pour l’analyse des données. Les données sont collectées pour l’assemblage d’affichages et d’interprétation personnalisés.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Paramètre de configuration de autoload</p></td>
<td align="left"><p>Paramètre de configuration de la stratégie d’exécution du client qui permet d’afficher automatiquement le bloc de fonctionnalité secondaire d’une application virtualisée au client en arrière-plan.</p></td>
</tr>
<tr class="even">
<td align="left"><p>brancher un package</p></td>
<td align="left"><p>Pour mettre à niveau un package d’application séquencé existant et l’exécuter côte à côte avec le package d’application séquencé d’origine.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Composition de suite dynamique</p></td>
<td align="left"><p>Une fonctionnalité de virtualisation des applications qui permet à un package d’application virtuelle d’autoriser les plug-ins dépendants ou les packages middleware à utiliser l’environnement virtuel. Cette fonctionnalité permet aux plug-ins et aux logiciels intermédiaires d’utiliser les paramètres de Registre du package principal, et les packages se comportent et interagissent de la même manière que dans le cas où ils étaient installés localement sur un dispositif d’activation du micro.</p></td>
</tr>
<tr class="even">
<td align="left"><p>Fichier ICO</p></td>
<td align="left"><p>Fichier de l’icône sur le Bureau du client utilisée pour lancer une application séquencée.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Racine de la source d’icône</p></td>
<td align="left"><p>Référence à une clé de Registre qui autorise la spécification d’un emplacement source pour la récupération d’icône pour un package d’application séquencé lors de la publication. Les racines de source d’icône prennent en charge les formats UNC uniquement (pas les URL).</p></td>
</tr>
<tr class="even">
<td align="left"><p>Répertoire d’installation</p></td>
<td align="left"><p>Le répertoire où le programme d’installation pour le Sequencer de la virtualisation de l’application place ses fichiers.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Client de bureau Microsoft Application Virtualization</p></td>
<td align="left"><p>Il s’agit d’une application qui se trouve sur un ordinateur de bureau Windows, qui communique et s’authentifie auprès du serveur d’applications virtuelles Microsoft System Center pour recevoir le code de l’application et permettre l’exécution locale d’une application séquencée.</p></td>
</tr>
<tr class="even">
<td align="left"><p>Microsoft Application Virtualization pour les services Terminal Server</p></td>
<td align="left"><p>La fonctionnalité de virtualisation des applications, notamment les composants client et serveur, qui s’exécutent dans un environnement de services Terminal Server.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Microsoft Application Virtualization Management Console</p></td>
<td align="left"><p>Composant logiciel enfichable centralisé à la console de gestion Microsoft qui vous permet d’administrer le système de gestion de la virtualisation des applications.</p></td>
</tr>
<tr class="even">
<td align="left"><p>Système de gestion de Microsoft Application Virtualization</p></td>
<td align="left"><p>Un déploiement spécifique de la plateforme de virtualisation des applications qui inclut tous les composants gérés par un magasin de données unique.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Service Web de gestion de Microsoft Application Virtualization</p></td>
<td align="left"><p>Service agissant en tant qu’intermédiaire entre la console de gestion des applications et le magasin de données. Tout d’abord, il authentifie ses utilisateurs, puis permet aux administrateurs autorisés de manipuler des données dans le magasin de données.</p></td>
</tr>
<tr class="even">
<td align="left"><p>Plateforme Microsoft Application Virtualization</p></td>
<td align="left"><p>Nom des produits Microsoft utilisés pour créer, stocker, distribuer et exécuter des applications virtualisées.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Sequencer Microsoft Application Virtualization</p></td>
<td align="left"><p>Application qui surveille et enregistre le processus d’installation et de configuration des applications de sorte qu’une application puisse être séquencée et exécutée dans l’environnement virtuel.</p></td>
</tr>
<tr class="even">
<td align="left"><p>Client Microsoft Application Virtualization Terminal Services</p></td>
<td align="left"><p>Une application qui se trouve sur un serveur Terminal Server et qui communique et s’authentifie auprès du serveur d’applications virtuelles Microsoft pour recevoir le code de l’application et permettre l’exécution d’une application séquencée localement.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Serveur de gestion de la virtualisation des applications Microsoft System Center</p></td>
<td align="left"><p>Un des deux types de serveurs de virtualisation d’application à partir desquels un package d’application séquencé peut être diffusé. En plus des packages d’application virtuelle de diffusion en continu, le serveur de gestion des applications virtuelles fournit d’autres services tels que la publication, la gestion, le rapport, etc.</p></td>
</tr>
<tr class="even">
<td align="left"><p>Serveur de diffusion de la virtualisation des applications Microsoft System Center</p></td>
<td align="left"><p>Un des deux types de serveurs de virtualisation d’application à partir desquels un package d’application séquencé peut être diffusé. Le serveur de diffusion diffuse les applications sur les ordinateurs clients uniquement et ne propose pas d’autres services tels que la publication, la gestion, le rapport, etc.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Ouvrir un fichier d’OSD (Open Software Descriptor)</p></td>
<td align="left"><p>Fichier XML demandant au client de récupérer l’application séquencée à partir du serveur de gestion/streaming d’applications, ainsi que l’exécution de l’application séquencée dans son environnement virtuel.</p></td>
</tr>
<tr class="even">
<td align="left"><p>Racine source de l’OSD</p></td>
<td align="left"><p>Référence à une clé de Registre qui autorise la spécification d’un emplacement source pour la récupération de fichier OSD pour un package d’application lors de la publication. Les racines sources de l’OSD prennent en charge les formats UNC uniquement (pas les URL).</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Répertoire racine du package</p></td>
<td align="left"><p>Le répertoire sur l’ordinateur de séquençage sur lequel sont installés les fichiers pour le package d’application séquencé. Ce répertoire existe également virtuellement sur l’ordinateur sur lequel une application séquencée sera diffusée.</p></td>
</tr>
<tr class="even">
<td align="left"><p>bloc de fonctions principal</p></td>
<td align="left"><p>Le contenu minimal d’un package d’application nécessaire pour qu’une application s’exécute. Le contenu du bloc de fonctionnalité principal est identifié lors de la phase d’application du séquençage et se compose généralement du contenu des fonctionnalités d’application les plus utilisées.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>bloc de fonctionnalité secondaire</p></td>
<td align="left"><p>Le reste du package d’application qui n’est pas contenu dans le bloc de fonctionnalités principal. Ce contenu est transmis au client à la demande à mesure que les fonctionnalités de l’application sont utilisées.</p></td>
</tr>
<tr class="even">
<td align="left"><p>application séquencée</p></td>
<td align="left"><p>Application surveillée par le Sequencer, réparties dans des blocs de fonctionnalités principaux et secondaires, transmise à l’aide d’un ordinateur exécutant le client Microsoft Application Virtualization Terminal Server ou du client de bureau Microsoft Application Virtualization et qui peut s’exécuter au sein de son propre environnement virtuel.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>package d’application séquencé</p></td>
<td align="left"><p>Fichiers comprenant une application virtuelle et autorisant l’exécution d’une application virtuelle. Ces fichiers sont créés après le séquençage et incluent spécifiquement les fichiers. OSD,. SFT,. sprj et. ico.</p></td>
</tr>
<tr class="even">
<td align="left"><p>séquençage</p></td>
<td align="left"><p>Processus de création d’un package d’application à l’aide du séquenceur de virtualisation d’application. Dans ce processus, une application est surveillée, ses raccourcis sont configurés et un package d’application séquencé est créé contenant les fichiers. OSD,. SFT,. sprj et. ico.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>séquençage d’un ordinateur</p></td>
<td align="left"><p>Ordinateur utilisé pour exécuter le séquençage et créer un package d’application séquencé.</p></td>
</tr>
<tr class="even">
<td align="left"><p>Assistant de séquencement</p></td>
<td align="left"><p>Assistant pas à pas qui guide l’utilisateur via le séquençage d’une application, y compris la configuration de package, l’installation de l’application ou des applications à des fins de séquence et le séquençage du package d’application pour la diffusion en continu.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>ASSOCIATIONS</p></td>
<td align="left"><p>Fichier qui contient une ou plusieurs applications séquencées que le Sequencer a empaquetées dans des blocs de flux continu, ainsi que les informations de remise associées. Le fichier SFT est stocké sur chaque serveur qui doit diffuser en continu les applications empaquetées sur un client.</p></td>
</tr>
<tr class="even">
<td align="left"><p>SFTMIME</p></td>
<td align="left"><p>Une interface de ligne de commande pour la configuration et la gestion des applications, des associations de types de fichiers et des serveurs de configuration de bureau gérées par les clients de bureau et de services Terminal Server.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Fichier SPRJ</p></td>
<td align="left"><p>Un fichier de projet de Sequencer XML dans lequel le Sequencer stocke ses éléments d’exclusion et analyse les informations d’éléments. Le fichier SPRJ est essentiellement utilisé pour créer des enregistrements d’application, ainsi que lors d’une mise à niveau d’un package.</p></td>
</tr>
<tr class="even">
<td align="left"><p>application virtuelle</p></td>
<td align="left"><p>Application empaquetée par le Sequencer pour s’exécuter dans un environnement virtuel autonome. L’environnement virtuel contient les informations nécessaires à l’exécution de l’application sur le client sans installer l’application localement.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>COM virtuel</p></td>
<td align="left"><p>Le sous-système qui gère les objets COM créés par les processus d’application exécutés dans un environnement virtuel et évite les conflits avec les mêmes objets créés en dehors de l’environnement virtuel.</p></td>
</tr>
<tr class="even">
<td align="left"><p>Répertoire virtuel</p></td>
<td align="left"><p>Un répertoire opaque dans lequel seuls les fichiers et sous-dossiers définis dans le package d’application ou créés via une interaction avec une application dans un environnement virtuel sont visibles. Tout fichier d’un répertoire local portant le même nom n’est pas visible pour l’application.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>environnement virtuel</p></td>
<td align="left"><p>Conteneur d’exécution qui définit les ressources disponibles pour les processus d’application lancés à partir d’un package d’application séquencé.</p></td>
</tr>
<tr class="even">
<td align="left"><p>fichier virtuel</p></td>
<td align="left"><p>Nom de fichier au sein de l’environnement virtuel mappé à un autre emplacement cible. Un fichier virtuel apparaît à côté des autres fichiers dans l’annuaire contenant, qu’il s’agisse d’un répertoire virtuel ou local.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>système de fichiers virtuel</p></td>
<td align="left"><p>Le sous-système qui intercepte et redirige les demandes de système de fichiers à partir de processus d’application s’exécutant dans un environnement virtuel. Ces requêtes sont traitées en fonction des fichiers et répertoires virtuels définis dans le package de l’application et créés ou modifiés par le biais de l’interaction avec une application virtuelle.</p></td>
</tr>
<tr class="even">
<td align="left"><p>Registre virtuel</p></td>
<td align="left"><p>Le sous-système qui intercepte et redirige les requêtes de Registre pour les clés et les valeurs de processus d’application exécutés dans un environnement virtuel. La redirection repose sur les informations de Registre définies dans le package d’application et créées ou modifiées par le biais d’une interaction avec une application virtuelle.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>services virtuels</p></td>
<td align="left"><p>Sous-système agissant en tant que gestionnaire de contrôle de service (SCM) pour les services exécutés dans un environnement virtuel.</p></td>
</tr>
</tbody>
</table>

# <span data-ttu-id="0515b-103">Nouveautés de MED-V2.0</span><span class="sxs-lookup"><span data-stu-id="0515b-103">What's New in MED-V 2.0</span></span>


<span data-ttu-id="0515b-104">Microsoft Enterprise Desktop Virtualization (MED-V) 2,0 a évolué la prise en charge de la compatibilité des applications pour Windows 7 et des fonctionnalités supprimées qui ne sont pas requises pour ce scénario.</span><span class="sxs-lookup"><span data-stu-id="0515b-104">Microsoft Enterprise Desktop Virtualization (MED-V) 2.0 has evolved the application compatibility support for Windows 7 and removed functionality that is not required for this scenario.</span></span> <span data-ttu-id="0515b-105">Par exemple, des fonctionnalités telles que le chiffrement de l’espace de travail MED-V et du serveur centralisé MED-V et de l’espace de travail MED-V ont été supprimés.</span><span class="sxs-lookup"><span data-stu-id="0515b-105">For example, features such as encryption of the MED-V workspace, the centralized MED-V server, and MED-V workspace trim transfer have been removed.</span></span>

## <span data-ttu-id="0515b-106">Modifications apportées aux fonctionnalités standard</span><span class="sxs-lookup"><span data-stu-id="0515b-106">Changes in Standard Functionality</span></span>


<span data-ttu-id="0515b-107">Cette section décrit les principaux domaines dans lesquels la fonctionnalité 2,0 MED-V a changé.</span><span class="sxs-lookup"><span data-stu-id="0515b-107">This section discusses the key areas where MED-V 2.0 functionality has changed.</span></span>

### <span data-ttu-id="0515b-108">Création d’espace de travail MED-V</span><span class="sxs-lookup"><span data-stu-id="0515b-108">MED-V Workspace Creation</span></span>

<span data-ttu-id="0515b-109">Le disque dur virtuel utilisé pour l’espace de travail MED-V est désormais créé dans l’ordinateur virtuel Windows.</span><span class="sxs-lookup"><span data-stu-id="0515b-109">The virtual hard disk used for the MED-V workspace is now created in Windows Virtual PC.</span></span> <span data-ttu-id="0515b-110">Les méthodes qui sont utilisées pour créer l’espace de travail MED-V incluent l’installation de Windows XP SP3, la mise à jour du système d’exploitation et la préparation de leur gestion via l’infrastructure de gestion des logiciels.</span><span class="sxs-lookup"><span data-stu-id="0515b-110">The methods that are used to create the MED-V workspace include installing Windows XP SP3, updating the operating system, and preparing it to be managed through software management infrastructure.</span></span>

<span data-ttu-id="0515b-111">La fonctionnalité gestion hors connexion et transfert de découpage a été supprimée, en plus des fonctionnalités de chiffrement et de compression d’espace de travail MED-V.</span><span class="sxs-lookup"><span data-stu-id="0515b-111">The offline management and trim transfer functionality were removed, in addition to the proprietary MED-V workspace encryption and compression functionality.</span></span> <span data-ttu-id="0515b-112">Lorsque vous créez un espace de travail MED-V, un administrateur doit préparer et configurer les applications et outils de gestion appropriés dans l’image au lieu d’utiliser l’outil de préparation de l’ordinateur virtuel fourni dans MED-V 1,0.</span><span class="sxs-lookup"><span data-stu-id="0515b-112">When you create a MED-V workspace, an administrator should prepare and configure appropriate applications and management tools in the image instead of using the virtual machine preparation tool that is provided in MED-V 1.0.</span></span>

<span data-ttu-id="0515b-113">L’exécution de Sysprep sur l’image MED-V est désormais obligatoire et validée lors de l’empaquetage de l’espace de travail MED-V.</span><span class="sxs-lookup"><span data-stu-id="0515b-113">Running Sysprep on the MED-V image is now required and validated during the packaging of the MED-V workspace.</span></span> <span data-ttu-id="0515b-114">Le module de création de package de l’espace de travail MED-V fournit une interface utilisateur graphique qui reguide l’administrateur dans le processus de création de packages.</span><span class="sxs-lookup"><span data-stu-id="0515b-114">The MED-V Workspace Packager provides a graphical user interface (GUI) that guides the administrator through the packaging process.</span></span> <span data-ttu-id="0515b-115">La console de MED-V 1,0 a été retirée en même temps que les fonctionnalités de gestion des images, de gestion des profils d’espace de travail MED-V et de la nécessité de mettre en place et de chiffrer les espaces de travail MED-V.</span><span class="sxs-lookup"><span data-stu-id="0515b-115">The console from MED-V 1.0 was removed together with the functionality of managing images, managing MED-V workspace profiles, and the requirement to stage and encrypt MED-V workspaces.</span></span>

### <span data-ttu-id="0515b-116">Déploiement d’espace de travail MED-V</span><span class="sxs-lookup"><span data-stu-id="0515b-116">MED-V Workspace Deployment</span></span>

<span data-ttu-id="0515b-117">Pour le déploiement d’un espace de travail MED-V, un administrateur peut désormais bénéficier de ses outils de distribution de logiciels électroniques.</span><span class="sxs-lookup"><span data-stu-id="0515b-117">To deploy a MED-V workspace, an administrator is now able to take advantage of their electronic software distribution tools.</span></span> <span data-ttu-id="0515b-118">La méthode de collecte client disponible dans MED-V 1,0 a été supprimée et l’espace de travail MED-V est désormais remis à l’aide de méthodes en dehors de MED-V.</span><span class="sxs-lookup"><span data-stu-id="0515b-118">The client-pull method available in MED-V 1.0 was removed and the MED-V workspace is now delivered by using methods outside MED-V.</span></span> <span data-ttu-id="0515b-119">Les administrateurs peuvent traiter les espaces de travail MED-V comme tout autre package d’application et pouvoir planifier des déploiements et des installations de MED-V en utilisant leurs outils et processus existants.</span><span class="sxs-lookup"><span data-stu-id="0515b-119">Administrators can treat MED-V workspaces as they would any other application package and can schedule deployments and installations of MED-V by using their existing tools and processes.</span></span> <span data-ttu-id="0515b-120">Les installations MED-V peuvent être déployées en silence et peuvent facilement être gérées au sein d’une infrastructure de distribution de logiciels existante.</span><span class="sxs-lookup"><span data-stu-id="0515b-120">MED-V installations can be deployed silently and can easily be managed inside an existing software distribution infrastructure.</span></span>

### <span data-ttu-id="0515b-121">Gestion de l’espace de travail MED-V</span><span class="sxs-lookup"><span data-stu-id="0515b-121">MED-V Workspace Management</span></span>

<span data-ttu-id="0515b-122">L’espace de travail MED-V dans MED-V 2,0 est basé sur un disque dur virtuel Windows Virtual PC.</span><span class="sxs-lookup"><span data-stu-id="0515b-122">The MED-V workspace in MED-V 2.0 is based on a Windows Virtual PC virtual hard disk.</span></span> <span data-ttu-id="0515b-123">MED-V a étendu les fonctionnalités fournies par le PC virtuel Windows en améliorant l’interface sans nécessiter un chiffrement ou un outil spécial pour accéder à l’espace de travail MED-V.</span><span class="sxs-lookup"><span data-stu-id="0515b-123">MED-V has extended the capabilities that Windows Virtual PC provides by improving the seamless experience without requiring encryption or special tools to access the MED-V workspace.</span></span>

<span data-ttu-id="0515b-124">Après le déploiement de MED-V sur une station de travail, l’espace de travail MED-V peut être ouvert en mode plein écran à l’aide de l’ordinateur virtuel Windows.</span><span class="sxs-lookup"><span data-stu-id="0515b-124">After MED-V is deployed to a workstation, the MED-V workspace can be opened in full-screen mode by using Windows Virtual PC.</span></span> <span data-ttu-id="0515b-125">Cette nouvelle fonctionnalité supprimait la configuration requise pour les stratégies qui ont défini une préférence pour les modes plein écran, et supprimée la nécessité de forcer l’affichage en plein écran des diagnostics et de la résolution des problèmes.</span><span class="sxs-lookup"><span data-stu-id="0515b-125">This new functionality removed the requirement for policies that set a preference for seamless or full-screen modes and also removed the need to force full-screen for diagnostics and troubleshooting.</span></span>

<span data-ttu-id="0515b-126">La publication d’applications sur l’espace de travail MED-V n’est plus effectuée avec les profils et en entrant manuellement le chemin d’accès aux applications.</span><span class="sxs-lookup"><span data-stu-id="0515b-126">Publishing applications to the MED-V workspace is no longer performed with profiles and by manually entering the path to applications.</span></span> <span data-ttu-id="0515b-127">Au lieu de cela, il se produit automatiquement lorsque des applications sont installées sur l’invité.</span><span class="sxs-lookup"><span data-stu-id="0515b-127">Instead, it occurs automatically as applications are installed on the guest.</span></span> <span data-ttu-id="0515b-128">Le référentiel d’image central qui inclut les versions des images fournies par le biais du transfert de découpage est supprimé.</span><span class="sxs-lookup"><span data-stu-id="0515b-128">The central image repository that included versions of the images that were delivered through trim transfer is removed.</span></span> <span data-ttu-id="0515b-129">En revanche, la gestion des applications et des mises à jour de MED-v permet aux administrateurs de gérer l’espace de travail MED-V de façon à ce qu’ils soient distribués sans la complexité d’une infrastructure MED-V dédiée.</span><span class="sxs-lookup"><span data-stu-id="0515b-129">Instead, MED-V enables administrators to manage the MED-V workspace as they would a physical computer, by letting applications and updates be distributed without the complexity of a dedicated MED-V infrastructure.</span></span>

## <span data-ttu-id="0515b-130">Modifications apportées aux fonctionnalités de MED-V</span><span class="sxs-lookup"><span data-stu-id="0515b-130">Changes in MED-V Features</span></span>


<span data-ttu-id="0515b-131">Plusieurs domaines clés de MED-V 2,0 reflètent les améliorations ou les ajouts apportées aux fonctionnalités suivantes.</span><span class="sxs-lookup"><span data-stu-id="0515b-131">Several key areas of MED-V 2.0 reflect improvements or additions to the following features.</span></span>

### <span data-ttu-id="0515b-132">Création d’espace de travail MED-V</span><span class="sxs-lookup"><span data-stu-id="0515b-132">MED-V Workspace Creation</span></span>

<span data-ttu-id="0515b-133">Les espaces de travail MED-V doivent être créés à l’aide du PC virtuel Windows.</span><span class="sxs-lookup"><span data-stu-id="0515b-133">MED-V workspaces must be created by using Windows Virtual PC.</span></span> <span data-ttu-id="0515b-134">Les images 2007 de PC virtuelles doivent être déplacées.</span><span class="sxs-lookup"><span data-stu-id="0515b-134">Existing Virtual PC 2007 images must be migrated.</span></span> <span data-ttu-id="0515b-135">L’outil virtual machine PReP n’est pas inclus dans MED-V 2,0 et les administrateurs doivent configurer, mettre à jour et optimiser leurs images conformément au fichier d’aide de MED-V 2,0.</span><span class="sxs-lookup"><span data-stu-id="0515b-135">The virtual machine Prep tool is not included in MED-V 2.0 and administrators should configure, update, and optimize their images according to the MED-V 2.0 Help file.</span></span> <span data-ttu-id="0515b-136">L’exécution de Sysprep sur l’image MED-V est une étape requise qui doit être effectuée avant l’empaquetage.</span><span class="sxs-lookup"><span data-stu-id="0515b-136">Running Sysprep on the MED-V image is a required step and must be performed before packaging.</span></span>

### <span data-ttu-id="0515b-137">Package d’espace de travail MED-V</span><span class="sxs-lookup"><span data-stu-id="0515b-137">MED-V Workspace Packaging</span></span>

<span data-ttu-id="0515b-138">Windows PowerShell est la base du gestionnaire de package d’espace de travail MED-V.</span><span class="sxs-lookup"><span data-stu-id="0515b-138">Windows PowerShell is the foundation of the MED-V Workspace Packager.</span></span> <span data-ttu-id="0515b-139">Cette fonctionnalité remplace certaines fonctions et fonctionnalités de la console et les fonctions centralisées de MED-V.</span><span class="sxs-lookup"><span data-stu-id="0515b-139">This functionality replaces some former console abilities and functionality that managed centralized functions of MED-V.</span></span> <span data-ttu-id="0515b-140">Le module de création de package de l’espace de travail MED-V compresse simplement le disque dur virtuel avec les paramètres et l’image appropriés pour pouvoir être déployé facilement par les administrateurs.</span><span class="sxs-lookup"><span data-stu-id="0515b-140">The MED-V Workspace Packager merely packages the virtual hard disk with the appropriate settings and image so that it can be easily deployed by administrators.</span></span> <span data-ttu-id="0515b-141">Les fonctionnalités avancées sont fournies à l’aide de Windows PowerShell.</span><span class="sxs-lookup"><span data-stu-id="0515b-141">Advanced features are provided by using Windows PowerShell.</span></span>

### <span data-ttu-id="0515b-142">Distribution d’espace de travail MED-V</span><span class="sxs-lookup"><span data-stu-id="0515b-142">MED-V Workspace Distribution</span></span>

<span data-ttu-id="0515b-143">L’infrastructure de serveur dédiée n’est plus nécessaire pour MED-V 2,0 et la méthode pull du client pour le déploiement d’espaces de travail MED-V a été supprimée.</span><span class="sxs-lookup"><span data-stu-id="0515b-143">Dedicated server infrastructure is no longer required for MED-V 2.0 and the client pull method to deploy MED-V workspaces was removed.</span></span> <span data-ttu-id="0515b-144">Les espaces de travail MED-V sont désormais déployés à l’aide de votre infrastructure de distribution de logiciels électroniques et peuvent être stockés sur des partages communs qui sont utilisés pour d’autres packages d’installation.</span><span class="sxs-lookup"><span data-stu-id="0515b-144">MED-V workspaces are now deployed using your electronic software distribution infrastructure and can be stored on common shares that are used for other installation packages.</span></span>

### <span data-ttu-id="0515b-145">Configuration pour la première fois</span><span class="sxs-lookup"><span data-stu-id="0515b-145">First Time Setup</span></span>

<span data-ttu-id="0515b-146">Le processus de configuration pour la première fois est désormais intégré à la Convention d’imagerie standard de Sysprep.</span><span class="sxs-lookup"><span data-stu-id="0515b-146">The first time setup process is now integrated with the standard imaging convention of Sysprep.</span></span> <span data-ttu-id="0515b-147">Le processus de configuration de l’espace de travail MED-V peut appliquer de manière dynamique des paramètres spécifiés dans le gestionnaire de package de l’espace de travail MED-V à l’image pendant qu’elle lance le mini-programme d’installation.</span><span class="sxs-lookup"><span data-stu-id="0515b-147">The MED-V workspace first time setup process can dynamically apply settings specified in the MED-V Workspace Packager to the image as it begins Mini-Setup.</span></span> <span data-ttu-id="0515b-148">L’outil de script de la console a été supprimé et le processus de configuration pour la première fois est désormais basé sur les options configurées dans le gestionnaire de package MED-V de l’administrateur.</span><span class="sxs-lookup"><span data-stu-id="0515b-148">The scripting tool in the console was removed and the first time setup process is now based on options that are configured in the MED-V Workspace Packager by the administrator.</span></span>

### <span data-ttu-id="0515b-149">Publication d’applications</span><span class="sxs-lookup"><span data-stu-id="0515b-149">Application Publishing</span></span>

<span data-ttu-id="0515b-150">Les administrateurs peuvent installer des applications sur l’image MED-V avant de procéder au conditionnement, après le déploiement de l’espace de travail MED-V ou à l’aide d’une combinaison des deux.</span><span class="sxs-lookup"><span data-stu-id="0515b-150">Administrators can install applications on the MED-V image either before packaging, after the MED-V workspace is deployed, or by using a combination of both.</span></span> <span data-ttu-id="0515b-151">MED-V n’examine plus la stratégie d’espace de travail MED-V pour publier des applications, mais désigne plutôt ce qui est réellement installé sur l’invité.</span><span class="sxs-lookup"><span data-stu-id="0515b-151">MED-V no longer examines MED-V workspace policy to publish applications, but instead refers to what is actually installed on the guest.</span></span> <span data-ttu-id="0515b-152">À mesure que des applications sont installées sur l’invité, celles-ci sont automatiquement détectées et publiées dans le menu de **démarrage** de l’hôte et sont prêtes à être démarrées par l’utilisateur final.</span><span class="sxs-lookup"><span data-stu-id="0515b-152">As applications are installed on the guest, they are automatically detected and published to the host **Start** menu and are ready to be started by the end user.</span></span>

### <span data-ttu-id="0515b-153">Redirection d’URL</span><span class="sxs-lookup"><span data-stu-id="0515b-153">URL Redirection</span></span>

<span data-ttu-id="0515b-154">MED-V 2,0 fournit une redirection d’une adresse Web d’hôte à invité transparente en fonction des stratégies configurées et gérées par l’administrateur.</span><span class="sxs-lookup"><span data-stu-id="0515b-154">MED-V 2.0 provides seamless host-to-guest web address redirection based on the policies configured and managed by the administrator.</span></span> <span data-ttu-id="0515b-155">Dès lors que l’URL est redirigée vers le navigateur invité, l’utilisation par défaut consiste à limiter l’utilisateur à ce site Redirigé.</span><span class="sxs-lookup"><span data-stu-id="0515b-155">After a URL is redirected to the guest browser, the default experience is to attempt to limit the user to that redirected site.</span></span> <span data-ttu-id="0515b-156">Cela permet de réduire les activités de navigation que l’utilisateur peut effectuer et qui ne sont pas prévues par l’administrateur.</span><span class="sxs-lookup"><span data-stu-id="0515b-156">This minimizes the browsing activities that a user can perform that are not intended by the administrator.</span></span> <span data-ttu-id="0515b-157">La redirection de navigateur invité à hôte a été supprimée.</span><span class="sxs-lookup"><span data-stu-id="0515b-157">Guest-to-host browser redirection was removed.</span></span>

of standard host-based processes for troubleshooting.</span></span> <span data-ttu-id="0515b-160">Dans la mesure où l’espace de travail MED-V n’est plus chiffré, il est possible de l’ouvrir en mode
